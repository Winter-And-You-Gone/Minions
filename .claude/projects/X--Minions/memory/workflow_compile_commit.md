---
name: 改代码后必须编译、提交、同步
description: 每次修改代码后都要本地重新编译，成功后提交并推送到 GitHub
type: feedback
---

每次修改代码后，必须执行以下流程：
1. **本地编译** — 运行项目的构建命令，确保编译通过
2. **提交** — 编译成功后，创建 git commit
3. **同步** — 推送到 GitHub 远程仓库

**Why:** 用户要求每次改动都保持可编译状态，并同步到 GitHub 以保证代码不丢失、可追溯。

**How to apply:** 在完成任何代码修改后，先执行编译命令，若编译失败则修复；成功后自动执行 git add → commit → push 流程。
